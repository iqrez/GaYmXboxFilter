5. Script issues

- install-driver.ps1 crashes with a null StackNames binding when the device is in a reboot-required disconnected/problem state. That should be handled as a warning or explicit reboot-required outcome, not as a PowerShell parameter-binding error.

- uninstall-driver.ps1 cleans DriverStore state correctly, but the broader operator workflow still leaves boot policy changes behind unless reverted separately.
